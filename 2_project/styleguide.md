# 风格指南
## 1. 术语
+ 翻译时所使用的术语词汇必须遵照所提供的**该产品标准词库**
+ 某些词汇在术语库中可能存在一种以上的释义，此时必须选用**最确切的译文**
+ 所有未在术语库中出现的其他词汇，如一些技术词，应充分**借助网络/书籍资源选择一个最合适的释义**
+ 必须保证译文的**术语一致性**
+ 如发现术语库/表中的术语释义有误，请及时反馈
## 2. 表达方式
+ 译文句子要简洁，易读，避免出现大量长句
+ 避免使用被动语态，会令用户难以快速理解信息
+ 避免使用俚语，小心使用口语，用对话的语气和用户沟通是可以接受的，但要保持专业。
+ 避免带有对性别、年龄、种族、职业、宗教信仰、政治信仰、政党、国籍、地区、贫富以及身体残疾者攻击性的用词及语气
+ 译文须遵循中文句法和表达习惯
+ 原则上，尽量少使用“不…就不…”或“不是不…”等双重否定的表示方法

    |Country code|国家代码|国家/地区代码|
    |Countries|各国|各国/各地区|
+ 简短单词和常用语列表
  
    |英文|中文翻译|中文使用场景|
    | :-----: | :-----: | :-----: |
    |app|应用|app-应用；application-应用程序|
    |PC|电脑|对于包含PC的术语，请遵循术语库/表中的标准翻译|
    |computer|计算机|对于包含computer的术语，请遵循术语库/表中的标准翻译|
    |pick|选取|pick-选取；select-选择|
    |attribute|属性|attribute-属性；property-特性|

+ 避免使用的单词和短语
  
    |英文|中文翻译（旧）|中文翻译（新）|
    | :-----: | :-----: | :-----: |
    |blog|日志|博客/微博|
    |if|倘若/假使/假若|假如/如果|
    |to|至|到|
    |this/these|此|这/这些|
    |if you want|若要|要/如果要|
    |only|仅是|只是|
+ 保留不译的内容
    + 商标、注册商标、服务标记不译。（商标符号和服务标记符号应标记为上标符号。）
    + 文档中的外国人名一般不译。
    + 文档中的生僻地名可不译，常见地名的翻译以中国地图出版社最新版的地图以及地名录为准。
    + 外国通信地址一般不译，中国地址应该上网查询，确定准确译法。（其中数字一律用阿拉伯数字，不用汉语写法。）
    + 卷名、文件名、文件扩展名、目录、设备名，路径及网址(URL)的名称不译。
    + 版权归作者所有，任何形式转载请联系作者。
    + 外国公司名一般不翻译，在文档中第一次遇到时保留英文，后括中文，如：SonyCorp.(Sony公司）。以后叙述中可只用括号内的译法。
    + Product ID、CD key以及Certificate of Authenticity如果在产品包装中只以英文出现，则文档中保留英文。
    + 属性名、方法名、事件名、保留字、参数的某些特定值不翻译，只译对值的解释。
    + 产品名不译。如：Windows, Windows 3.1等。
    + 操作键不译，保留英文。如Tab、Esc、Enter等。
    +  Internet、Intranet、WWW和Web不译。
    +  文档中引用的英文书名，如果没有中文译本或本地化，则保留英文。
    +  文档中引用其他未本地化的产品或文档时，提及的界面用语、标题请保留英文。或使用格式：English (中文翻译）。
    +  所有命令行、参数、命令开关选项、程序代码等不译。
    + 中文中有未翻译的英文单词或缩写时，若原文为直接加“s”的复数形式，则应该删去“s”而将其还原为单数形式。
> ### 2.2 句子的表达
+ You做指称时，一律译成“你”，无需译成“您”。
+ 祈使句的翻译不用译出主语。
  
  比如：Let apps give you personalized content based on your PC’s location, name, account picture, and other domain info. 

  译成：允许应用根据你的电脑位置、姓名、用户头像及其他域信息为你提供个性化内容。 

  避免译成：你可以...
+ 被动句的翻译首选省略人称的译法。
  
  比如：All 3 records were created successfull
  
  译成：3 条记录都已创建成功。 
  
  避免译成：你已成功创建3条记录。
## 3.格式的统一
> ### 3.1 字间距
+ 全角中文字与半角英文字/数字之间必须有一个半角宽度的空格；
+ 所有全角标点符号与前后字符之间不留空格；
+ 半角标点符号与前后字符（除全角标点）间留有空格，按照英文的书写原则处理。
> ### 3.2标点符号
    针对软件和文档的本地化有两套不同的标点符号规范，具体内容如下。
+ 逗号
  
  中文逗号用于软件和文档本地化。注意：英文中列举几个项目时使用逗号，而中文中使用顿号。
+ 冒号
  
  中文冒号后无需空格。
+ 破折号和连字符
  
    + 短破折号（负号、减号）前后个包含一个空格
    +  中文中避免使用长破折号(—)，防止与汉字“一”难以区分。
+ 省略号

  中文省略号为六个点（……）
+ 引号
    + 单引号在双引号内使用，其余情况一律使用双引号
  
        比如：请查阅第一章的“运行 Setup 出问题时查阅‘诊断和解决问题’一章”的内容。
    + 包含需要点击的内容时，用双引号注明要点击的内容
  
        比如：单击“文件”选项卡。
+ 括号
    
    如果括号中的文本包含双字节字符，则使用中文括号（）。中文括号和括号外的文本之间没有空格。如果括号中的文本是单字节字符和/或数字，则应该使用英文括号()。在英文括号和括号外的文本之间应该留有单字节空间。
+ 方括号
  
  中文翻译一律使用英文方括号（单字节方括号）[]。
